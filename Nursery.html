Nursery Dataset#

Nursery Dataset merupakan kumpulan data yang digunakan untuk melatih dan mengevaluasi model pembelajaran mesin dalam klasifikasi. Dataset ini berasal dari sistem pengambilan keputusan hierarkis yang dikembangkan untuk memeringkat aplikasi ke sekolah taman kanak-kanak di Ljubljana, Slovenia pada tahun 1980-an. Nursery atau sering kita kenal sebagai lembaga pendidikan untuk anak-anak berusia 2 hingga 4 tahun. Untuk mengetahui lebih dekat dengan dataset ini, di bawah telah disiapkan pemaparan sekilas terkait dataset tersebut, di antaranya:

No Nama Fitur Type Deskripsi Value
1 parents ordinal sikap orangtua terhadap pendidikan anak usual, pretentious, great_pret
2 has_nurs ordinal ketersediaan perawat di rumah proper, less_proper, improper, critical, very_crit
3 form ordinal kelengkapan formulir pendaftaran penitipan anak complete, completed, incomplete, foster
4 children ordinal jumlah anak dalam keluarga 1, 2, 3, more

Data Understanding#

Data understanding merupakan sebuah tahapan awal di dalam metodologi sains data dan pengembangan AI yang bertujuan untuk mendapatkan pemahaman awal mengenai data yang dibutuhkan untuk memecahkan permasalahan yang ada. Dimana, sering kali data dari berbagai sumber belum tentu dapat langsung digunakan karena:

  1. Maksud dan tujuan data berbeda-beda

  2. Keadaan asal terpisah-pisah atau justru terintegrasi secara ketat

  3. Tingkat kekayaan (richness) berbeda-beda

  4. Tngkat keandalan (reliability) berbeda-beda

Dalam kasus ini, kita akan memahami data tentang Nursery Dataset. Dimana langkah-langkah yang diterapkan akan dijelaskan di bawah ini dengan telah dipisahkan antar sub-bab.

Convert Dataset to CSV#

Langkah selanjutnya merupakan convert dataset yang semula dari online ke offline(file local) berupa ekstensi .csv. Hal ini berguna untuk berbagai hal, di antaranya untuk back up file jika kedepannya ada kendala pada dataset yang ada, untuk mempermudah pelacakan data yang ada di dalamnya, dan banyak lagi. Berikut ini merupakan implementasi code untuk convert dataset ke ekstensi .csv.

import pandas as pd

# Mengubah data ke DataFrame
nursery_df = pd.DataFrame(nursery.data.features)

# Menambahkan kolom target
nursery_df['target'] = nursery.data.targets

# Menyimpan data ke file .csv
nursery_df.to_csv('nursery.csv', index=False)

print("Dataset telah disimpan dalam file 'nursery.csv'")

Dataset telah disimpan dalam file 'nursery.csv'

Menarik Dataset#

Kemudian, menarik dataset dari berbagai sumber yang ada. Hal ini berguna untuk analisis dan eksplorasi data. Pada case ini, dibuatkan contoh saat menarik data dari Github dan MySQL. Keduanya adalah dataset yang sama, namun berbeda dalam cara memanggil/menarik data tersebut saja.

  • Menarik data dari Github

Pertama, menarik data dari Github. Maksudnya, kita menyimpan file dataset yang sebelumnya telah di convert ke .csv ke dalam repository Github. Berikut ini penjelasan beserta langkah singkat untuk step-by-step menarik data dari Github.

  1. Buatlah repository baru.

  2. Tempatkan file dataset tersebut pada directory bebas (bisa di dalam folder atau langsung filenya).

  3. Jika sudah, buka file tersebut.

  4. Lalu, pilih opsi ‘Raw’ yang terletak bar kanan atas (sisi kiri ‘download’).

  5. Kemudian, copy URL pada field search engine yang digunakan.

  6. Tinggal, pemanggil dataset pada umumnya. pd.read_csv(‘paste url disini’).

Implementasi langkah-langkah tersebut ke code Python adalah sebagai berikut.

Berikutnya, menarik data dari MySQL. Hal ini kurang lebih sama konsepnya dengan cara sebelumnya (lewat Github). Akan tetapi, dalam langkah ini agak sedikit kompleks jika dilakukan oleh orang awam atau kurang familiar dengan hal ini. Untuk langkah-langkahnya adalah sebagai berikut.

  4. Kemudian, Connection sesuai step 2. Dan dilanjut menambah ‘New Database’.

Algoritma ini didasarkan pada Teorema Bayes dan asumsi kemerdekaan fitur bersyarat, yang berarti bahwa nilai satu fitur tidak bergantung pada nilai fitur lain.

Cara Kerja Naive Bayes:

  1. Hitung probabilitas apriori: Hitung probabilitas setiap kelas dalam dataset.

  2. Hitung probabilitas kondisional: Hitung probabilitas setiap fitur untuk setiap kelas.

  3. Terapkan Teorema Bayes: Gunakan Teorema Bayes untuk menghitung probabilitas posterior dari setiap kelas untuk data baru.

  4. Prediksi kelas: Pilih kelas dengan probabilitas posterior tertinggi sebagai prediksi untuk data baru.

Algoritma ini bekerja dengan mencari K tetangga terdekat dari data baru dalam ruang fitur dan menggunakan mayoritas kelas dari tetangga tersebut sebagai prediksi kelas data baru.

Cara Kerja KNN:

  1. Tentukan nilai K: Pilih nilai K, yang merupakan jumlah tetangga terdekat yang akan digunakan untuk prediksi.

  2. Hitung jarak: Hitung jarak antara data baru dan setiap data dalam dataset.

  3. Identifikasi tetangga terdekat: Temukan K data dalam dataset yang memiliki jarak terdekat dengan data baru.

  4. Prediksi kelas: Hitung mayoritas kelas dari K tetangga terdekat. Kelas yang paling banyak muncul di antara tetangga terdekat akan menjadi 5. prediksi kelas untuk data baru.

Ensemble Learning#

Ensemble learning adalah sebuah teknik dalam machine learning di mana beberapa model (biasanya disebut sebagai “base learners” atau “weak learners”) digabungkan untuk meningkatkan kinerja prediksi dibandingkan dengan penggunaan model tunggal. Ide dasar di balik ensemble learning adalah bahwa dengan menggabungkan berbagai model yang mungkin memiliki kelemahan yang berbeda, kita dapat menghasilkan model yang lebih kuat dan lebih stabil secara keseluruhan. Berikut ini penjelasan bagian-bagian dari tipe Ensemble Learning yang dipelajari pada mata kuliah ditempuh (Penambangan Data) beserta implementasinya.

  • Stacking

Stacking melibatkan penggabungan beberapa model berbeda dan menggunakan model yang lebih tinggi (meta-model) untuk menggabungkan hasil prediksi dari model-model tersebut. Stacking memungkinkan untuk memanfaatkan kekuatan berbagai jenis model. Stacking bekerja dengan melatih dua model:

  1. Model Dasar: Melatih beberapa model dasar (misalnya, Naive Bayes, KNN) pada dataset.

  2. Model Meta: Melatih model meta (misalnya, Naive Bayes) untuk memprediksi kelas data baru berdasarkan prediksi dari model dasar.

Cara Kerja Stacking:

  1. Latih Model Dasar: Latih beberapa model dasar pada dataset dan dapatkan prediksi mereka untuk setiap data.

  2. Gabungkan Prediksi: Gabungkan prediksi dari model dasar menjadi dataset baru, di mana setiap data diwakili oleh vektor prediksi dari model dasar.

  3. Latih Model Meta: Latih model meta pada dataset prediksi yang digabungkan untuk mempelajari hubungan antara prediksi model dasar dan kelas target.

  4. Prediksi Baru: Gunakan model meta yang terlatih untuk memprediksi kelas data baru berdasarkan prediksi dari model dasar.

import pandas as pd
from sklearn.naive_bayes import GaussianNB
from sklearn.ensemble import BaggingClassifier
from sklearn.metrics import accuracy_score

# Path dataset
X_train_el = "nursery_train_encoded.csv"
y_train_el = "nursery_train_target_encoded.csv"
X_test_el = "nursery_test_encoded.csv"
y_test_el = "nursery_test_target_encoded.csv"

# Load data as DataFrames
X_train = pd.read_csv(X_train_el)
y_train = pd.read_csv(y_train_el)['target']
X_test = pd.read_csv(X_test_el)
y_test = pd.read_csv(y_test_el)['target']

# Base Learner: Naive Bayes
base_clf = GaussianNB()

n_estimators_list = [25, 55, 85, 115, 145]  # Daftar n_estimators

for n_estimators in n_estimators_list:
    # Buat model BaggingClassifier dengan base learner dan n_estimators
    bagging_clf = BaggingClassifier(base_estimator=base_clf, n_estimators=n_estimators)

    # Fit model pada data training
    bagging_clf.fit(X_train, y_train)

    # Prediksi pada data testing
    y_pred = bagging_clf.predict(X_test)

    # Evaluasi model
    accuracy = accuracy_score(y_test, y_pred)

    print(f"\n~n_estimators: {n_estimators}~")
